Q1

unsqueeze를 쓰지 않고 열 벡터를 뽑았을 때 .shape 차이점은?

예제 비교

x[:, 0]           # unsqueeze 없이
x[:, 0].unsqueeze(1)  # unsqueeze 사용

1. x[:, 0]

  • 결과 텐서 shape: torch.Size([3])
  • 설명: 1열의 값을 1차원 벡터로 반환한다. 예: [0.6614, -0.4519, -1.0276]

2. x[:, 0].unsqueeze(1)

  • 결과 텐서 shape: torch.Size([3, 1])
  • 설명: 열 방향으로 2차원 텐서로 만든다. 예: [[0.6614], [-0.4519], [-1.0276]]

차이점 요약

항목 x[:, 0] x[:, 0].unsqueeze(1)
shape [3] [3, 1]
차원 수 1차원 (벡터) 2차원 (행렬)
브로드캐스팅 자동으로 확장 어려움 다른 행렬과 곧바로 연산 가능

언제 unsqueeze(1)를 써야 하나?

  • 모델에 입력할 때 (예: [batch, features] 형태)
  • 행렬 연산에서 차원 일치를 맞춰야 할 때
  • 시각화 시 열 벡터 형태로 유지하고 싶을 때

Q2

x[:, 1:3] vs. x[:, [1, 2]] 차이점

1. x[:, 1:3] — 슬라이싱

  • 연속된 구간을 slice 객체로 추출
  • 결과는 연속된 메모리 블록이므로 뷰(view)가 됨 → 원본 변경 시 반영 가능
  • 메모리 효율적
  • 인덱스는 start:end로, end는 포함하지 않음
x[:, 1:3]  # 열 1번과 2번

2. x[:, [1, 2]] — Fancy Indexing

  • 리스트로 지정한 인덱스들을 직접 선택
  • 복사본(copy)이 생성되므로 원본 변경과 연결되지 않음
  • 연속된 열뿐 아니라 임의의 순서, 중복, 불연속 인덱스도 가능
x[:, [1, 2]]       # 열 1번과 2번 (복사본)
x[:, [2, 1, 2]]    # 열 순서, 중복도 가능

차이 예시

항목 x[:, 1:3] x[:, [1, 2]]
연속된 열만 선택 가능 ✅ ✅
불연속 열 선택 ❌ ✅
중복 열 선택 ❌ ✅ ([2, 1, 2] 등)
반환값이 뷰(view)? ✅ ❌ (copy)
메모리 효율 높음 낮음 (복사 발생)

Q3

x[1:3, :] vs. x[-2:, :] 관계

동일한 결과를 만듦

둘 다 2~3행(인덱스 1, 2)을 선택하는 코드다.

x[1:3, :]    # 행 인덱스 1~2
x[-2:, :]    # 끝에서 두 번째부터 끝까지 = 인덱스 1~2
표현 의미 행 추출 결과
x[1:3, :] 1번째부터 2번째까지 슬라이싱 [1, 2]행
x[-2:, :] 뒤에서 2개 슬라이싱 [1, 2]행

차이점

  • x[1:3, :]은 정방향 인덱스로 위치를 기준
  • x[-2:, :]은 역방향 인덱스로 끝 기준 → 가변 길이 텐서에서 유용

Q2 Q3 정리 포인트

비교 대상 차이점 요약
[:, 1:3] vs [:, [1, 2]] 전자는 연속 구간 슬라이싱, 후자는 지정된 열 복사
[1:3, :] vs [-2:, :] 결국 같은 행 추출, 접근 방식(앞 vs. 뒤 기준)만 다름

Q4

unsqueeze에 대해 설명

  • unsqueeze는 텐서 차원을 인위적으로 추가하는 함수로, PyTorch에서 데이터를 다룰 때 매우 자주 쓰인다.
  • 차원이 맞지 않아 오류가 발생할 때, unsqueeze를 이용하면 이를 간단하게 해결할 수 있다.

torch.unsqueeze(tensor, dim) 또는 tensor.unsqueeze(dim)

역할:

  • 지정한 위치 dim에 크기 1짜리 차원을 추가한다.
  • 결과 텐서의 shape은 원래보다 차원이 1 증가한다.

예시 1: 벡터 → 열 벡터

x = torch.tensor([1, 2, 3])       # shape: [3]
x_unsq = x.unsqueeze(1)           # shape: [3, 1]

# 결과:
# tensor([[1],
#         [2],
#         [3]])

예시 2: 벡터 → 배치 텐서

x = torch.tensor([1, 2, 3])       # shape: [3]
x_unsq = x.unsqueeze(0)           # shape: [1, 3]

# 결과:
# tensor([[1, 2, 3]])

Shape 변화 정리:

원래 shape unsqueeze(0) unsqueeze(1) unsqueeze(-1)
[3] [1, 3] [3, 1] [3, 1]
[3, 4] [1, 3, 4] [3, 1, 4] [3, 4, 1]

주로 사용하는 상황

1. 모델 입력 차원 맞추기

x = torch.tensor([1.0, 2.0, 3.0])  # shape [3]
x = x.unsqueeze(0)  # shape [1, 3] → batch 차원 추가

2. 브로드캐스팅 용도

a = torch.tensor([1, 2, 3])         # shape: [3]
b = a.unsqueeze(1)                  # shape: [3, 1]

→ 이후 연산 시 자동으로 [3, 1]과 [3, 4] 같은 텐서가 맞춰짐

주의

  • unsqueeze는 실제 데이터를 바꾸지 않고 shape만 변경한다.
  • 원래의 텐서와 동일한 메모리 참조를 가질 수도 있음 (즉, view일 수 있음)

관련 함수

함수 기능
unsqueeze 차원 추가
squeeze 크기 1인 차원 제거
reshape 전체 차원을 원하는 형태로 재구성
view reshape과 거의 동일, 메모리 연속 요구 
torch.unsqueeze?
Docstring:
unsqueeze(input, dim) -> Tensor
Returns a new tensor with a dimension of size one inserted at the
specified position.
The returned tensor shares the same underlying data with this tensor.
A :attr:`dim` value within the range ``[-input.dim() - 1, input.dim() + 1)``
can be used. Negative :attr:`dim` will correspond to :meth:`unsqueeze`
applied at :attr:`dim` = ``dim + input.dim() + 1``.
Args:
    input (Tensor): the input tensor.
    dim (int): the index at which to insert the singleton dimension
Example::
    >>> x = torch.tensor([1, 2, 3, 4])
    >>> torch.unsqueeze(x, 0)
    tensor([[ 1,  2,  3,  4]])
    >>> torch.unsqueeze(x, 1)
    tensor([[ 1],
            [ 2],
            [ 3],
            [ 4]])
